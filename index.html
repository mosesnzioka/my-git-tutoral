What is Git

Git is a Version Control System.

A version control system is a software that tracks and manages changes to files over time.
Version Control Systems allow users to revisit earlier versions of their files, compare changes between versions,
and undo changes.

Other version control systems are:_

  • Subversion
  • CVS
  • Mercurial

They all have a common goal but for now we will be talking about Git and GitHub.

What is GitHub

GitHub is a web-based platform that provides version control and collaboration tools for software development.
It allows developers to host, review, and manage code, track changes, and collaborate with others on projects.
GitHub is built around Git, making it easier for teams to work together and maintain a history of their codebase.

Installing Git

First of all confirm if Git is installed into your computer and to do this follow following steps.

  • On the command prompt you open before type git --version and enter key.
  • If a version number is returned, Git is installed on your computer. If you receive any other output, Git is not installed.
  • If not installed use this link to download Git version on your computer.download Git!

Configuring Git

To configure Git does as follows

  • To configure username type git config --global user.name "USERNAME" and press enter key.
  • To configure email type git config --global user.email "EMAIL" and press enter key.
  • For example if your username is daniel, type git config --global user.name "daniel" and press enter key.
    and if your email is example@gmail.com type git config --global user.email "example@gamil.com" and press enter key.
  • Rember that you should have a github account becouse the email and username should be as they appear in your github account.

Initializing Github Repository

What is Git Repository

A git Repository is a workspace which tracks and manages files within a folder.
A point to note a project should have a single Git repository.

Before initializing your git repository type git status to confirm if was initialized before becouse it should be initialized just once.
When you run the git status command and get the message "fatal: not a git repository..." being returned,
then it means that the project does not have a git repository, you can go ahead and initialize one

To initialize Git repository go to your command prompt and type git init and press enter key.

Adding files to the staging Area

Staging area is an intermediate space where you can organize changes before committing them to the repository.
When you make changes to files in your working directory, those changes aren't immediately included in the next commit.
Instead, you need to add these changes to the staging area using the git add . command.

in Git there are two files which are Tracked and Untracked, and the difference between this files is that:_

  • Tracked files:git is aware of and is tracking for changes
  • Untracked files: These are files that exist in your working directory but have not been added to the repository

Committing file

A commit in git is basically a snapshot of your project at a particular point in time.
when writing a commit message you should ask your self this questionwhat this commit will do...?
the you write the answer as the commit message. To commit files you use this command git commit -m "message"

To go back to previous commit and going to specifict commit you use this commandsgit logand
git log --oneline respectively.


Branches in Git

Creating and Viewing branches in Git

Branch in Git is a separate workspace where you can work on new features, bug fixes,
or experiments without affecting the main codebase. Branches are important becouse they anable colaboration.

To creat a new Branch in your Git you use this command git branch BRANCH_NAME. To view branch you use this command git branch
always the active branch will be colored. For example lets create a branch called moses_nzioka and view it.

Switching active Branch

To switch branch you type git switch BRANCH_NAME to your command prompt. For example lets switch to the branch we created before.

Merging Branch

Almost all the times, we want to incorporate things we've done in our branches into destination branch.

To merge branch:

  • First, we switch to the branch where we want to merge the current branch. Then use this commandgit merge BRANCH_NAME
  • where BRANCH_NAME is the name of the branch we want to merge to the current active branch.
  • Lets assum our official branch is called 'main' and we have content in a branch called 'moses_nzioka' that we want to merge into main
  • We start by switching to the main branch where we want to merge the changes from 'moses_nzioka'git switch main .
  • We then use the git merge to merge the content from moses_nzioka to main git merge moses_nzioka .

Deleting Branch

After working on branches and merging the changes to the main branch, usually we don't need that branch anymore,
and it is safe to delete it.
To delete a branch, we use the git branch command and pass the -d flag git branch -d BRANCH_NAME
